# Authorization & RBAC

## `@RequiresRole(roles)`

The simplest form of authorization — require the authenticated user to have one of the given roles:

```ts
@RequiresRole('admin')
@Delete('/:id')
async deletePet(@Param('id') id: string) {
  return this.petService.delete(id);
}
```

Requests from a user without a matching role are rejected before the handler runs.

## `@Protect` — fine-grained access control

For anything more granular than "has role X," RapidREST has a built-in access-control-list (ACL) system. `@Protect` exists at two levels:

**On a model**, to protect every record of that type:

```ts
@Protect(
  {
    uid: 'Pet',
    records: [
      {userOrRoleId: 'anonymous', create: false, read: true, update: false, delete: false, special: false, full: false},
      {userOrRoleId: '.*', create: false, read: true, update: false, delete: false, special: false, full: false},
    ],
  },
  true, // recordACL: give every new Pet its own overridable per-record ACL
)
@DataStore('mongo')
export class Pet extends BaseMongoEntity {}
```

**On a route or handler**, to protect specific endpoints directly:

```ts
@Route('/admin')
@Protect({records: [{userOrRoleId: '.*', create: false, read: false, update: false, delete: false, special: false, full: false}]})
export class AdminRoute {}
```

### The ACL shape

```ts
interface ACLRecord {
  userOrRoleId: string; // "anonymous", ".*" (any authenticated user), a specific uid, or a role name
  create: boolean | null;
  read: boolean | null;
  update: boolean | null;
  delete: boolean | null;
  special: boolean | null;
  full: boolean | null; // true supersedes every other field
}
```

Each incoming request is matched against `records` in order — by exact user id, then `.*`/`*` (any authenticated user), then role membership — and the corresponding action (`create`/`read`/`update`/`delete`, derived from the HTTP method) must be `true` for the request to proceed.

The second `@Protect` argument, `recordACL`, controls whether **individual records** get their own ACL (inheriting from the model's class-level ACL as a parent, but independently overridable) — use this when different records of the same type need different permissions, e.g. a document that's been explicitly shared with a specific user.

## Turning it off, and trusted roles

```ts
conf.defaults({
  rbac: {enabled: true},
  trusted_roles: ['admin'],
});
```

`rbac:enabled` is a global switch — set it to `false` to bypass every ACL check (useful in tests). Users with any role listed in `trusted_roles` bypass ACL checks entirely, regardless of what the record's ACL says.
