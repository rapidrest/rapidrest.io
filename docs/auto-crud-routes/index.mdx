# Auto CRUD Routes

Given a model, `CRUDRoute<T>` generates a full set of REST endpoints for it — no hand-written handlers required.

```ts
import {CRUDRoute, RouteDecorators} from '@rapidrest/service-core';
import {Pet} from '../models/Pet.js';

const {Model, Route} = RouteDecorators;

@Model(Pet)
@Route('/pets')
export class PetRoute extends CRUDRoute<Pet> {}
```

That's the entire route class. `@Model(Pet)` tells the generated CRUD machinery which entity to operate on; the entity itself must have `@DataStore(...)` set (see [Datastores & Connections](/docs/models-persistence/datastores)) so RapidREST knows where to persist it.

## Generated endpoints

| HTTP | Path | What it does |
|---|---|---|
| `GET` | `/pets` | Find all matching objects (supports query filtering) |
| `HEAD` | `/pets` | Count matching objects — result in the `Content-Length` header |
| `POST` | `/pets` | Create one or more new objects |
| `PUT` | `/pets` | Bulk-update multiple objects |
| `DELETE` | `/pets` | Truncate — delete all matching objects |
| `GET` | `/pets/:id` | Find a single object by id |
| `HEAD` | `/pets/:id` | Check existence — `Content-Length` is `1` or `0` |
| `PUT` | `/pets/:id` | Update a single object (optimistic locking via `version`) |
| `DELETE` | `/pets/:id` | Delete a single object (supports `?version=` and `?purge=true`) |
| `PUT` | `/pets/:id/:property` | Update a single property directly, bypassing the full-object optimistic-lock update |

:::note[The "me" shortcut]
Every endpoint that takes an `:id` treats the literal value `me` as shorthand for the authenticated user's own `uid`. `GET /users/me` returns the current user without the caller needing to know their own id.
:::

Every generated endpoint is also reflected in the automatically-generated OpenAPI spec — see the API documentation served at `/openapi.json`.

## `ModelRoute` vs. `CRUDRoute`

`CRUDRoute<T>` is what you extend to get every endpoint above out of the box. It extends a lower-level `ModelRoute<T>`, which provides the underlying behaviors (`doFind`, `doFindById`, `doCreate`, `doUpdate`, `doDelete`, `doCount`, `doTruncate`, `doUpdateProperty`, ...) but registers **no HTTP endpoints itself**. Extend `ModelRoute` directly instead of `CRUDRoute` when you only want to expose a subset of the standard endpoints.

## Overriding a single endpoint

Extend `CRUDRoute`, override the one method you need to customize, and call the parent implementation:

```ts
@Model(User)
@Route('/users')
export class UserRoute extends CRUDRoute<User> {
  @Get('/:id')
  @After('cleanPII')
  findById(@Param('id') id: string, @Query() query: any, @User user?: any) {
    return super.findById(id, {query, user});
  }

  private cleanPII(user: User, @User caller?: any) {
    if (!caller) {
      user.email = undefined;
    }
    return user;
  }
}
```

This is the standard pattern: keep every generated endpoint, but post-process the response for the one that needs it — see [Lifecycle & Validation](/docs/routing-controllers/lifecycle-validation) for more on `@After`.

## Exposing only some endpoints

Extend `ModelRoute` directly and wire up only the decorators you want:

```ts
@Model(Pet)
@Route('/pets')
export class ReadOnlyPetRoute extends ModelRoute<Pet> {
  @Get()
  find(@Query() query: any) {
    return super.doFind({query});
  }

  @Get('/:id')
  findById(@Param('id') id: string) {
    return super.doFindById(id, {});
  }

  // No create/update/delete endpoints are registered at all.
}
```
