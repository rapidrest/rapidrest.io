# Core Concepts

Three ideas underpin everything else in RapidREST. Understanding them first makes the rest of the framework predictable rather than "magic":

1. **[Dependency Injection](/docs/core-concepts/dependency-injection)** — how `@Inject`, `@Config`, and `@Logger` get properties populated, and how `@Init`/`@Destroy` hook into an object's lifecycle.
2. **[Auto-Discovery](/docs/core-concepts/auto-discovery)** — how every class you export under `src/` ends up registered and injectable, with no central registration file.
3. **[Configuration](/docs/core-concepts/configuration)** — the nconf-based config system and its colon-separated key paths, which almost everything else in the framework reads from.
