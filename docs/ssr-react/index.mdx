# SSR React

`@rapidrest/react` renders React pages on the server using a file-convention `app` directory, in the spirit of Next.js, but wired directly into RapidREST's own server and dependency injection, with no second framework, dev server, or build pipeline to run alongside it.

- **[Getting Started](/docs/ssr-react/getting-started)**: install the package, write a layout and a page, and mount it.
- **[App Directory Convention](/docs/ssr-react/app-convention)**: `_layout`, `_404`/`_500`, and how URLs map to page files.
- **[Data Fetching](/docs/ssr-react/data-fetching)**: `fetchProps`, and injecting server-side services into a page.
- **[Dev Mode](/docs/ssr-react/dev-mode)**: the `rapidreact` CLI and live reload.
